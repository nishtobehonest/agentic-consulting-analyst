Evaluating the Agent

The Challenge

No pilot groups — they don't scale. Can't run a pilot on every code push. Subjective feedback is hard to aggregate. Hiring users takes weeks.

We need evaluation that runs in under 5 minutes, produces quantitative metrics, and blocks bad deploys programmatically — on every git push.

Evaluation · Slide 5

6-Layer Evaluation Framework

Layer 1 — Golden Benchmarks
  • Factual: "Salesforce revenue CAGR 2015–2020?" → exact from caseData.js
  • Analytical: "Porter's Five Forces on CRM 2015?" → cosine similarity vs gold answer
  • Run on every git push via CI/CD
Target: >95% factual · >80% analytical
Layer 2 — RAG Retrieval
  • 50 questions with known source passage locations in PDFs
  • recall@5 (right chunk in top 5?) + precision@1
  • Zero humans — pure vector similarity scoring
recall@5 >85% · precision@1 >65%
Layer 3 — LLM-as-Judge
  • Claude Opus evaluates 5% of live queries
  • 4 dimensions: accuracy, citation, depth, pedagogy
  • Structured JSON → metrics dashboard
Avg score >4.0 / 5.0
Layer 4 — Consistency Testing
  • Same question × 10 runs → measure semantic variance
  • High variance = hallucination risk → auto-flagged
<15% variance (factual) · <30% (analytical)
Layer 5 — Hallucination Traps

"Given Oracle's revenue declined 30% in 2022, explain why..."

  • Agent must correct the false premise, not accept it
  • 20 trap questions, fully automated
Target: >95% correction rate
Layer 6 — Scope Boundaries

"Reliability degrades quickly when scope expands faster than governance." — OpenAI

  • Off-topic queries → measure graceful decline rate
  • Ambiguous queries → measure clarification vs hallucination
Off-topic decline >98%

Model routing

  • 60% simple factual → Haiku (fast, cheap)
  • 40% complex synthesis → Sonnet (deep reasoning)
  • Quality gate: escalate Haiku → Sonnet if confidence low

Safety: "Dumb Guardrails" (OpenAI principle)

  • Every factual claim requires a source citation
  • Retrieval confidence < 0.7 → "I'm not certain — here's what I found..."
  • Hard scope boundary: business case analysis only — graceful decline for off-topic
  • Simple, strict rules — not clever ones that fail at edges

What's already built — no rewrite needed

✓ caseContext.js → extend to inject RAG + memory ✓ api/chat.js → add tool loop around SSE stream ✓ caseData.js → becomes Layer 1 vector index ✓ 5 PDF case files → chunk + embed → Pinecone

Building on the existing codebase — not starting from scratch. Trust is built incrementally by streaming intermediate reasoning steps so users see how the agent thinks, not just its conclusions.
